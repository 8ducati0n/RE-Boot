# 0. 요약 (1쪽)

## 무엇을 만들었는가
**부트캠프형 SW·AI 교육의 중도 이탈률 감소와 AI 과의존 방지를 위한 Human-in-the-Loop 적응형 학습 플랫폼 'RE:Boot'**

## 왜 만들었는가
- 부트캠프 중도 이탈률 **약 21%**, 1인당 국비 손실 **최대 2,000만 원** (박진아·김지은, 2024)
- AI 기반 교육 도구 확산에 따른 학습자의 **AI 과의존** 리스크 (EDM, 2025)
- 두 문제는 별개가 아니라 **메타인지 실패의 두 얼굴**이라는 가설

## 어떻게 풀었는가 — 4 모듈, 진단 → 개입 → 보정 순환
1. **수준 진단 + 갭 맵** (Vygotsky ZPD)
2. **형성평가 + 간격 반복** (Bloom 완전학습 + Ebbinghaus 망각곡선)
3. **AI-TPACK HITL 협업** (AI 제안 → 교수자 승인)
4. **조기경보 + 동료 그룹핑** (Learning Analytics)

## 구현 단위 — 학습자 5 컴포넌트 + 강사 F1
- F1: 강사 커리큘럼 등록 칸
- 학습자 0: 전체 과정 이수율 (상시)
- 학습자 1: 학습목표 자기평가 (수업 전)
- 학습자 2: 체크리스트 (수업 후, 증거 입력형)
- 학습자 3: Socratic CAM LLM (실습·자습 중)
- 학습자 4: 망각곡선 퀴즈 pop-up (수업 후 며칠 뒤)

## 검토 요청 핵심
- **기획 설계**: 이론 적용의 정확성, 4 모듈 통합 구조의 학술적 타당성, 메타인지 4장치의 인지심리학적 근거, HITL 역할 분리의 윤리·교육적 정당성
- **기술 구현**: 학습자·교수자 흐름의 사용성, Socratic 즉답 차단의 부담 수준, 증거 입력 체크리스트의 실효성, 강사 Composition Assistant의 실사용성, 정보 윤리·접근성

# 1. 제출 초록 (공식 anchor)

> 2026 한국교육정보미디어학회 춘계학술대회 발표신청서에 제출된 초록 원문입니다. 본 패키지의 모든 후속 자료는 이 초록과 충돌하지 않도록 작성되었습니다.

## 제목
**부트캠프형 SW·AI 교육의 중도 이탈률 감소와 AI 과의존 방지를 위한 Human-in-the-Loop 적응형 학습 플랫폼 'RE:Boot' 설계 및 개발**

## 초록 본문
국내 부트캠프형 SW·AI 교육의 중도 이탈률은 약 21%에 이르며(박진아·김지은, 2024), 이에 따른 1인당 국비 손실은 최대 2,000만 원에 달한다. 동시에 AI 기반 교육 도구의 확산과 함께 학습자의 AI 과의존 문제가 주요 리스크로 부각되고 있다(EDM, 2025). 본 연구는 이 두 문제를 동시에 다루기 위해 AI의 확장성과 교수자 판단의 신뢰성을 결합한 Human-in-the-Loop 적응형 학습 플랫폼 'RE:Boot'를 설계·구현하였다. RE:Boot는 다음 네 가지 핵심 모듈을 **'진단 → 개입 → 보정'**의 순환 구조로 통합한다.

1. **첫째**, Vygotsky의 근접발달영역(ZPD) 이론에 기반한 수준 진단 및 갭 맵을 통해 학습자 간 사전지식 편차를 정량화한다.
2. **둘째**, Bloom의 완전학습 모형과 Ebbinghaus 망각곡선을 접목한 형성평가 및 간격 반복 시스템을 통해 지식 정착을 지원한다.
3. **셋째**, AI-TPACK 프레임워크에 따라 AI가 교육적 개입을 제안하고 교수자가 이를 승인하는 협업 모델을 적용하여, AI 과의존을 구조적으로 방지한다.
4. **넷째**, 학습분석 기반 조기경보 시스템과 데이터 기반 동료 학습 그룹핑을 통해 이탈 위험 학습자를 선제적으로 지원한다.

본 플랫폼은 AI의 모든 교육적 제안이 교수자 승인을 거쳐야만 학습자에게 전달되는 구조를 채택함으로써, 확장 가능하면서도 신뢰할 수 있는 AI 교육 환경을 실현하고자 한다.

# 2. 기획 설계 핵심

## 2-1. 진단 → 개입 → 보정 순환 구조

```
   ┌────────── 진단 ──────────┐
   │  Vygotsky ZPD 갭 맵       │
   └─────────────┬────────────┘
                 ↓
   ┌────────── 개입 ──────────┐
   │  AI-TPACK 협업 (HITL)     │
   │  + 조기경보 + 그룹핑       │
   └─────────────┬────────────┘
                 ↓
   ┌────────── 보정 ──────────┐
   │  Bloom 완전학습           │
   │  + Ebbinghaus 망각곡선    │
   └─────────────┬────────────┘
                 ↓
            (다음 진단으로)
```

## 2-2. 시스템 역할 구조 (HITL)

| 역할 | 담당 | 금지 영역 |
|---|---|---|
| **AI** | 지식 전달 · 학습자 행동·학습 데이터 수집·분석 · Socratic 방식 질문 응답(직답 회피) | 정서적 위로, 완성 메시지 생성, 평가 단어 사용 |
| **교수자(인간)** | AI 산출물의 선택과 검증(HITL) · 학습자에 대한 정서적 지원 · 분석 데이터 기반 개입 의사결정 | — |
| **학습자** | 자기 위치·도달 수준의 메타적 인식 · 인지적 어려움 구간 통과 | — |

→ AI의 **모든** 교육적 제안은 교수자 승인을 거쳐야만 학습자에게 전달된다.

## 2-3. 메타인지 착각 완화 4장치

| 시점 | 장치 | 핵심 원리 |
|---|---|---|
| 수업 전 | 체크리스트 · 학습목표 · 전체 과정 이수율 점검 | 자기 위치·도달 수준의 메타적 인식 |
| 수업 중 (질문 응답) | Socratic AI | 즉답 대신 역질문으로 사고 과정 유도 |
| 수업 중 (인지 활성) | CAM (Cognitive Activation Mechanism) | 유창한 답변이 만드는 능력 착각 차단 |
| 수업 외 | 교수자 정서 개입 (데이터 기반) | 인지적 어려움 구간에서 강사가 직접 개입 |

## 2-4. 4 모듈 ↔ 5 컴포넌트 ↔ 4장치 크로스워크

| 초록 4 모듈 | 학습자 5 컴포넌트 | 메타인지 4장치 |
|---|---|---|
| ① 수준 진단 + 갭 맵 | 0) 이수율 · 1) 학습목표 (수업 전) | 수업 전 |
| ② 형성평가 + 간격 반복 | 2) 체크리스트 (수업 후) · 4) 망각곡선 퀴즈 | — (지식 정착) |
| ③ AI-TPACK HITL 협업 | 3) Socratic CAM LLM + 강사 F1 | 수업 중 (Socratic + CAM) |
| ④ 조기경보 + 동료 그룹핑 | (강사 대시보드 + 동료 그룹핑) | 수업 외 (정서 개입) |

## 2-5. 다층적 가치 사슬

- **교육적 가치(학습자)**: 메타인지 착각 완화 → 학습 지속성 → 어려움 구간 통과율
- **운영 가치(기관)**: 중도 이탈률 감소 → 1인당 최대 2,000만 원 국비 손실 절감
- **사회적 가치(시스템)**: 훈련 데이터 기반 적합 직무 매칭 → 일자리 순환 안정성

## 3-3. 핵심 인터랙션 특성

1. **즉답 차단**: Socratic LLM은 학습자가 코드와 막힌 지점을 진술하기 전까지 답을 제공하지 않는다. `[Articulating] → [Coaching] → [Scaffolding-Hint → Principle → Knowledge → Modeling]` graduated sequence를 엄격히 유지한다.
2. **증거 입력 강제**: 체크리스트는 작성 코드 한 줄·실행 결과를 입력해야 완료된다. 미입력 항목은 자습 큐로 자동 큐잉된다.
3. **사실만 정리**: 교수자 대시보드의 AI 출력은 사실(facts) + 화제 옵션(topic options) + 강사 본인 과거 표현(expression materials)에 한정된다. 완성 메시지는 절대 생성하지 않는다.
4. **하위 25% 가드**: 이수율 비교 정보는 하위 25% 학습자에게는 노출이 약화된다(자기효능감 보호).
5. **적응형 망각곡선**: 정답률 80%↑ → 다음 간격 연장, 50%↓ → 단축 + 강사 대시보드 사실 알림.

# 5. 검토 요청 항목 — 기획 설계 (8개)

각 항목에 대해 **1) 동의 여부 / 2) 보강 의견 / 3) 위배되는 선행 연구가 있다면 출처**를 함께 남겨주시면 감사합니다.

| 번호 | 검토 항목 | 핵심 질문 |
|---|---|---|
| **P-1** | 4 모듈 통합의 학술적 타당성 | ZPD 진단 → AI-TPACK 개입 → Bloom·Ebbinghaus 보정의 순환이 단일 플랫폼으로 결합되는 것이 적절한가? 기존 통합 프레임워크와 어떻게 차별화되는가? |
| **P-2** | ZPD 이론 적용의 정확성 | 수준 진단 + 갭 맵을 Vygotsky ZPD로 정당화하는 것이 원전 해석에 부합하는가? 사회적 상호작용 차원이 빠지지는 않았는가? |
| **P-3** | Bloom + Ebbinghaus 결합의 학습과학적 근거 | 완전학습(mastery learning)과 간격 반복(spaced repetition)의 결합이 SW·AI 학습 도메인에서 유의한 효과가 있다는 선행 연구가 충분한가? |
| **P-4** | AI-TPACK 프레임워크 적용의 적절성 | "AI가 제안 → 교수자 승인"이 AI-TPACK이 의도한 교수자 역량 강화와 정합하는가, 아니면 교수자를 단순 검수자로 축소하는가? |
| **P-5** | 메타인지 4장치의 인지심리학적 근거 | 체크리스트·이수율·Socratic·CAM이 능력 착각(Dunning-Kruger 계열) 완화에 학술적으로 정당한가? 측정 가능한가? |
| **P-6** | HITL 역할 분리 (AI 인지 · 교수자 정서) | LLM의 정서적 발화를 금지하고 강사에게만 정서를 배정하는 정책이 교육적·윤리적으로 정당한가? 학습자가 강사가 부재한 시간대(자습)에 정서 지원을 받지 못하는 위험은 어떻게 다루는가? |
| **P-7** | 동료 그룹핑의 협력학습 정합성 | 데이터 기반 동료 그룹핑이 협력학습(CSCL) 연구의 그룹 구성 원칙과 정합하는가? 이질성/동질성 중 어떤 원칙을 채택했고 그 근거는 무엇인가? |
| **P-8** | KAEIM 미디어전 학술적 기여도 | 본 작업이 한국교육정보미디어학회 발표 기준의 학술적 기여(이론·방법·사례·시사점)에서 충분한 무게를 가지는가? 부족한 부분이 있다면 어디인가? |

# 6. 검토 요청 항목 — 기술 구현 (8개)

각 항목에 대해 **1) 사용성 평가(5점 척도) / 2) 구체적 개선 의견 / 3) 우선순위(High/Mid/Low)**를 남겨주시면 감사합니다.

| 번호 | 검토 항목 | 핵심 질문 |
|---|---|---|
| **T-1** | F1 강사 등록 → 5 컴포넌트 자동 생성 흐름 | 강사 입력 부담이 합리적인가? 입력 양식이 SW 부트캠프 실제 운영자에게 자연스러운가? |
| **T-2** | Socratic CAM LLM 즉답 차단 UX | 학습자가 답답해서 이탈하지 않을 적정 임계점은 어디인가? graduated sequence(Hint→Principle→Knowledge→Modeling) 완화 조건이 충분한가? |
| **T-3** | 체크리스트 증거 입력 요구의 학습자 부담 | "코드 한 줄·실행 결과 한 줄" 입력이 과도하지 않은가? 학습 의지가 낮은 학습자에게도 작동하는가? |
| **T-4** | 망각곡선 퀴즈 pop-up 타이밍 | 1·3·7·16일 간격이 부트캠프(6개월) 일정과 조화되는가? pop-up 빈도가 방해적이지 않은가? |
| **T-5** | 강사 대시보드 "사실만" 정책 | Composition Assistant가 사실+화제 옵션+표현 재료만 제공하는 정책이 강사의 실제 업무 부담을 감당 가능 수준으로 낮추는가? 완성 메시지를 생성하지 않는 것이 강사에게 과한 짐인가? |
| **T-6** | 하위 25% 학습자 비교 정보 약화 가드 | 비교 정보 약화 임계점(평균 -8%p)이 적절한가? 약화가 오히려 학습자에게 "분류된 느낌"을 주지는 않는가? |
| **T-7** | 데이터 수집·활용 정보 윤리 | 학습자 행동 데이터(사전·사후 자기평가, 체크리스트 증거, CAM 대화)를 수집·저장·분석하는 절차에 동의·삭제·열람권 등이 적절히 설계되어 있는가? |
| **T-8** | 접근성 (시각·인지·언어) | 색대비, 키보드 네비게이션, 모바일 대응, 한국어 외 언어 학습자 지원, 인지 부담 등 접근성 측면의 개선점은 무엇인가? |
