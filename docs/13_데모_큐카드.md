# 13. 부스 데모 큐 카드 — 3분 30초 라이브 walkthrough

**용도**: 미디어전 부스에서 관람객에게 라이브 데모 시연 시 사용하는 큐 카드
**작성일**: 2026-05-14
**원안**: `00_미디어전_마스터.md` 2절 (본 문서는 부스 무대 운영용으로 구체화)

> 본 큐 카드는 노트북 옆에 인쇄해 두고, 시연 시 한 눈에 다음 화면·발화·강조 포인트를 확인하는 용도다.

---

## 1. 시연 전 준비 (관람객 도착 전)

- [ ] `npm run dev` 서버 실행 (`web/` 폴더에서)
- [ ] 브라우저 첫 화면: **`/`** (홈)에서 대기
- [ ] 모든 페이지 한 번씩 클릭해 캐시 워밍 (이수율 → 자기평가 → 튜터 → 체크리스트 → 망각곡선 → 커리큘럼 → 강사)
- [ ] 화면 밝기 최대, 배율 100% 또는 110%
- [ ] 와이파이 연결 확인. 약하면 즉시 모바일 핫스팟으로 전환
- [ ] 큐 카드(이 문서) 인쇄본 노트북 좌측에 비치
- [ ] 명함 + 1쪽 요약본 5-10부 비치

## 2. 시연 진입 멘트 (관람객 접근 시)

**관람객이 부스에 멈춰 섰을 때**:

> "안녕하세요. 부트캠프 중도 이탈과 AI 과의존을 같이 다루는 학습 플랫폼 RE:Boot입니다. **3분 정도 시연** 보여드릴까요?"

→ 동의 시 진행. 거절 시 1쪽 요약본 건네고 인사.

## 3. 3분 30초 시연 큐 카드

### CUE 1. 홈 (`/`) — 0:00~0:20 (20초)

**화면**
- 첫 화면 그대로

**말할 내용**
> "RE:Boot는 부트캠프 중도이탈 21%와 AI 과의존, 두 문제를 같은 메커니즘으로 다룹니다. AI가 분석하고, 교수자가 결정합니다 — 이게 핵심 메시지입니다."

**손동작**
- 한 줄 카피 가리키기

**전환**: 상단 Navbar의 "이수율" 클릭

---

### CUE 2. 이수율 (`/gap-map`) — 0:20~0:50 (30초)

**화면**
- 부트캠프 진행률 막대 + 동기 평균 마커

**말할 내용**
> "학습자는 본인의 진행률을 항상 봅니다. 그런데 단순 진행률이 아니에요. **동기 평균 대비 본인 위치**를 함께 보여줍니다. 객관적 위치 인식 → 메타인지 보정의 출발점입니다."

**강조 포인트**
- "동기 평균" 마커를 손가락으로 짚기
- "**하위 25% 학습자에게는 비교 정보가 약화**됩니다. 자기효능감 보호 가드예요." → "동기 평균 보기" 토글 버튼 클릭해서 시연

**전환**: Navbar "자기평가" 클릭

---

### CUE 3. 자기평가 (`/placement`) — 0:50~1:20 (30초)

**화면**
- 수업 전 사전 평가 폼

**말할 내용**
> "수업 전 학습목표 자기평가입니다. 3단계로 본인 상태를 표시합니다."

**손동작**
- "전혀 모름 / 들어봤음 / 할 수 있음" 옵션 가리키기
- 사전 평가 빠르게 채우고 "수업 시작" 클릭

**강조 포인트**
- "수업 후 동일 항목을 다시 평가합니다. **사전 0/3 → 사후 3/3으로 뛰는 격차가 능력 착각의 흔적**입니다. 강사 대시보드에 자동 알림됩니다."

**전환**: Navbar "CAM 튜터" 클릭

---

### CUE 4. CAM 튜터 (`/tutor`) — 1:20~2:00 (40초) ★ 핵심 시연

**화면**
- 좌측: CAM 2-Stage Cycle + 7 Guidelines + Mode 분기 + 역할 분리 사이드바
- 우측: 채팅 빈 화면

**말할 내용**
> "여기가 핵심입니다. 학습자가 코드를 업로드하면 AI가 어떻게 반응하는지 보세요."

**손동작**
- 좌측 사이드바 "G6 ★ 함수 return None (3-Step)" 시나리오 버튼 클릭

**스크립트 재생 중 강조**
> "보세요. 답을 안 줍니다.
> 첫 응답은 **[Verbalize]** — '제가 본 것은 ~입니다'로 시작합니다.
> 다음은 **[Ground Truth]** — 잘못 본 부분이 있는지 사용자에게 확인합니다.
> 그 다음 **[Pass]** — 사용자에게 질문으로 공을 되돌립니다.
> 이게 CAM의 G6 ★ 3-Step입니다. 환각도 차단하고, 즉답도 차단합니다."

**Vending Mode 추가 시연 (시간 여유 시)**
- "Vending" 버튼 클릭 → "Vending 모드 시연" 시나리오 재생
- "사용자가 빠른 답이 필요할 때는 Vending Mode로 전환합니다. AI가 '전환합니다' 한 줄 알리고 즉답합니다."

**강조 포인트**
- 좌측 사이드바 "정서 위로 금지" 박스 짚기
- "**AI는 인지만**. 학습자가 정서적으로 힘들어 보이면 '강사 면담 요청' 옵션이 자동 노출됩니다."

**전환**: Navbar "체크리스트" 클릭

---

### CUE 5. 체크리스트 (`/checklist`) — 2:00~2:25 (25초)

**말할 내용**

**손동작**
- 첫 항목 체크 후 증거 칸을 가리킴
- "여기에 작성한 코드 한 줄을 직접 입력해야 합니다. 안다는 느낌과 실제 수행을 분리하는 장치입니다."

**전환**: Navbar "망각곡선" 클릭

---

### CUE 6. 망각곡선 (`/quiz`) — 2:25~2:50 (25초)

**말할 내용**

**강조 포인트**

## 4. 데모 중 자주 받는 질문 빠른 답변

| Q | 빠른 답 (1줄) |
|---|---|
| "이거 누가 만들었나요?" | 연세대 교육공학 석사과정 김혜진이 단독 설계·개발한 미디어전 출품작입니다. |
| "실제로 어디 부트캠프에 쓰이나요?" | 현재는 설계·구현 단계입니다. 실 도입은 IRB 통과 후 진행 예정입니다. |
| "AI가 답을 안 주면 답답하지 않아요?" | Mentor Mode에서 즉답을 차단하지만, 학습자가 빠른 답이 필요할 때는 Vending Mode로 즉시 전환됩니다. 강사 대시보드에 모드 전환 사실이 알림됩니다. |
| "강사가 모든 메시지를 직접 쓰는 게 부담 아닌가요?" | 자동 통과 항목(퀴즈 스케줄·체크리스트 발송)과 수동 검토(메시지·결정)를 분리합니다. |
| "오픈소스인가요?" | GitHub에 공개되어 있습니다. (QR 코드 가리킴) |
| "데이터는 어떻게 보호하나요?" | 본 데모는 합성 데이터입니다. 실 도입 시 4분류 세분화 동의 + 외부 LLM 학습 송신 금지 정책을 적용합니다. |

## 5. 시연 사고 대응

| 사고 | 대응 |
|---|---|
| **와이파이 끊김** | 모든 v2.1-C 페이지는 자기완결 데모로 작동합니다. CAM 튜터 스크립트도 로컬. 패닉 X. |
| **CAM 시나리오 재생 중 멈춤** | "초기화" 버튼 클릭 후 다른 시나리오로 전환. "이번엔 스코프 시나리오 보여드릴게요." |
| **화면 인식 안 됨** | HDMI 어댑터 재연결. 그동안 1쪽 요약본 건네면서 구두로 설명. |
| **관람객이 너무 많아 시간 없음** | CUE 4(CAM 튜터)만 1분 시연. 핵심 메시지는 "AI가 인지, 교수자가 정서". |
| **관람객이 회의적 ("그래서 효과는?")** | "측정 변인 4개를 정의했습니다. V-1 즉답 요구 빈도, V-2 회고 깊이, V-3 응용 정답률, V-4 대화-체크리스트 일치도. 별도 자료에 정리해두었어요." (`09_측정변인.md`) |
| **교수자가 평가하러 오심 ("학술적 기여는?")** | "초록의 4 모듈을 단일 사이클로 통합하고, 학습자 5 컴포넌트 + 강사 F1으로 구현했습니다. 자세한 설계 문서를 보내드릴까요?" → 명함 교환 |
