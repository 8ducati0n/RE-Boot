# 01. RE:Boot 서비스 정체성

> 본 문서는 RE:Boot가 **정확히 어떤 서비스인가**를 정의한다. 이후 모든 설계·UI·문서의 기준점.

---

## 📌 한 문장 정의

> **"부트캠프 학습자의 중도 이탈률을 낮추고 AI 과의존을 구조적으로 방지하는 Human-in-the-Loop 적응형 학습 플랫폼"**
>
> AI가 모든 학습 데이터를 분석해 제안하고, **교수자가 교육적 판단을 승인**한 뒤에야 학습자에게 전달되는 **2-Tier Trust 구조**를 채택한다.

이것은 LMS가 아니다. "**교육적 의사결정의 책임 배분 아키텍처**"를 구현한 프로토타입이다.

---

## 🎯 타깃 사용자

| 사용자 | 규모 가정 | 핵심 Job-to-be-Done |
|---|---|---|
| **① 부트캠프 학습자** (25~35세, 전직/비전공자) | 30~100명/기수 | "내 수준에 맞는 학습 경로로 취업까지 완주하고 포트폴리오 확보" |
| **② 부트캠프 교수자** | 1~3명/기수 | "수십 명을 개인화 지원하기 위한 AI 기반 증폭 도구" |
| ③ 매니저/운영책임자 | 1명/조직 | "이탈 위험 학습자 조기 식별 + 개입 효과 측정" | > 의사결정에 도움을 줄 것으로 예상됨 !

## 🧩 해결하는 두 가지 문제

### 문제 1: 부트캠프 중도 이탈률 21%

- **출처**: 박진아·김지은 (2024). 부트캠프형 소프트웨어 교육 인식과 학습 이탈 방지 요인에 대한 질적 연구. *컴퓨터교육학회 논문지*, 27(1).
- **원인 (F10/F4/F11/F3/C7 지표)**:
  1. 사전지식 편차 (C7, 9.71%)
  2. 동료 관계 부재 (F10, **17.65% — 1위**)
  3. 성취감 부족 (F4, 11.76%)
  4. 운영자 개입 타이밍 놓침 (F11, 10.59%)
  5. 커리큘럼 불일치 (F3, 8.24%)
- **비용**: 1인당 국비 최대 2,000만원 손실 → **사회적 비용**

### 문제 2: AI 과의존 (Over-reliance)

- **출처**: EDM 2025 Best Paper (Tang & Bosch). *Trust but Risk: Over-reliance on AI Educational Tools*.
- **현상**: 학습자가 AI 도구(ChatGPT 등)를 무비판적으로 사용 → 학습 효과 저하
- **리스크**: AI가 잘못된 정보를 제공해도 학습자가 걸러내지 못함
- **구조적 한계**: 교육적 맥락 판단까지 AI가 대체 시도 → 교수자 역할 축소

### 두 문제는 **반대 방향**으로 해결되기 쉽다

| 해결 방향 | 이탈률 | AI 과의존 |
|---|---|---|
| AI로 더 많은 개입 | ↓ | ↑ |
| AI 사용 축소 | ↑ | ↓ |

**RE:Boot의 구조적 답**: AI의 **확장성**과 교수자의 **신뢰성**을 2-Tier로 분리하면 둘 다 동시에 해결.

---

## 🏛️ 기존 접근과의 차이

| 서비스 카테고리 | 방식 | 한계 |
|---|---|---|
| 전통 LMS (Moodle, Canvas, Blackboard) | 모든 학습자에게 동일 콘텐츠, 교수자 수동 관리 | 확장 불가, 개인화 부재 |
| MOOC (Coursera, edX) | 단방향 영상 + 자동 채점 퀴즈 | 이탈률 오히려 높음, 피드백 부재 |
| AI 직결 (ChatGPT 학습용) | 학생이 AI에 직접 질문 | 환각·과의존, 교수자 역할 소멸 |
| 적응형 학습 (Khan Academy, Duolingo) | 알고리즘 기반 문제 난이도 조정 | 이론 기반 약함, 협동학습 부재 |
| **RE:Boot** | **AI 분석 + 교수자 승인 + 이론 기반 설계** | — |

## 🔧 서비스의 본질 흐름

```
┌─────────────────── 학습자 데이터 ───────────────────┐
│ (퀴즈 · 출석 · 펄스 · 오답 · 강의 참여 · 포트폴리오) │
└─────────────────────────┬───────────────────────────┘
                          │
                    ┌─────▼─────┐
                    │ AI 감지 + │
                    │ 제안 생성 │
                    └─────┬─────┘
                          │
              ┌───────────┴──────────┐
              │                      │
        ┌─────▼─────┐          ┌─────▼─────┐
        │  Tier 1   │          │  Tier 2   │
        │ Grounding │          │  교수자    │
        │  Check    │          │  검증     │
        │  (자동)   │          │  (HITL)   │
        └─────┬─────┘          └─────┬─────┘
              │                      │
              └───────────┬──────────┘
                          │
            AIRecommendation(APPROVED)
                          │
                    ┌─────▼─────┐
                    │  학습자   │
                    │  전달     │
                    └─────┬─────┘
                          │
                    학습자 반응
                          │
                    (순환 피드백 루프)
                          │
                          ▼
                  AI 모델 개선 학습
```

**이게 서비스의 전부다**. 35개 기능은 이 파이프라인을 떠받치는 부속품.

## 🎪 핵심 기능 6개 (MVP)

| # | 기능 | 사용자 행동 | 내부 동작 |
|---|---|---|---|
| **1** | **수준 진단 + 갭맵** | 진단 테스트 응시 → 갭맵 확인 | 정답률 기반 Lv1/2/3 판정 → 카테고리별 보유/갭/학습중 분류 → SVG 도넛 시각화 |
| **2** | **적응형 커리큘럼 + 교수자 승인** | 자동 생성된 커리큘럼 확인 → 학습 시작 | AI가 갭맵 기반 생성 → `AIRecommendation(PENDING)` → 교수자 검토 → 학습자 전달 |
| **3** | **형성평가 + 간격 반복** | 강의 수강 → 형성평가 → 오답 자동 복습 | FormativeResponse → SpacedRepetitionItem 동적 스케줄 (난이도·정답률 적응) |
| **4** | **Agentic RAG AI 튜터** | 강의 중 질문 → 출처 표시된 답변 | Query Analysis → HyDE → Multi-Query → Hybrid Retrieval → Reranking → CRAG → CoT Generation → Grounding Check → Self-Reflection → Follow-up |
| **5** | **조기경보 + 동료 그룹핑** | 위험 알림 → 교수자 그룹 편성 | AtRisk 감지 → 유사 약점 학습자 클러스터링 → GROUP_STUDY 제안 |
| **6** | **AI 제안 관리 탭 (교수자)** | 대시보드에서 PENDING 확인 → 승인/교체/거부 | AIRecommendation 큐 UI, 사유·증거 노출 |

## 🧭 서비스가 "아닌 것"

혼동을 막기 위해 명시:

- ❌ **LMS 대체재가 아님** — 기존 LMS에 부가되는 "AI 개입 계층"
- ❌ **AI 튜터 서비스가 아님** — AI 튜터는 6개 기능 중 하나일 뿐
- ❌ **학습 관리 대시보드가 아님** — 대시보드는 교수자 의사결정 지원 수단
- ❌ **범용 교육 플랫폼이 아님** — 부트캠프(성인 SW·AI 단기 교육)에 특화
- ❌ **완성된 상용 제품이 아님** — 교육공학 석사과정 프로토타입 + 학회 미디어전 출품작

## 🎓 학술적 포지셔닝

이 프로젝트는 **교육공학(Instructional Technology) 연구 프로토타입**이다. 산업용 제품이 아니며, 평가 기준은 다음과 같다:

| 평가 기준 | RE:Boot 기여 |
|---|---|
| **이론 기반 설계** | 5개 교육학 이론을 하나의 플랫폼에 통합 구현 |
| **Research Gap 해소** | HCAP/I-TPACK 이론을 실제 동작하는 시스템으로 구현 |
| **Human-AI Co-Adaptation** | CHI/AIED/EDM 2025 트렌드를 단일 아키텍처로 통합 |
| **실물 배포 경험** | 오라클 클라우드 배포 경험 (기존 `re_boot/` 레포) |
| **학회 대주제 정합성** | "확장성 + 신뢰성"을 2-Tier Trust로 직접 구현 |

---

## 📝 요약 카드 (심사위원 1분 버전)

> **RE:Boot은 부트캠프 학습자의 21% 이탈률과 AI 과의존 문제를 동시에 해결하기 위해 설계된 적응형 학습 플랫폼입니다.**
>
> **핵심 차별점**: AI가 학습 데이터를 분석하고 제안을 생성하되, 모든 교육적 결정은 교수자가 최종 승인해야 학습자에게 전달되는 2-Tier Trust 구조를 채택합니다.
>
> **이론 통합**: Vygotsky ZPD(진단), Bloom 완전학습·Ebbinghaus 망각곡선(보정), AI-TPACK(개입), Learning Analytics(관계 지원) — 5개 교육학 이론이 각각 하나의 구현 모듈과 1:1 매핑됩니다.
>
> **학회 대주제 대응**: "확장성"은 AI의 자동 분석·생성으로, "신뢰성"은 교수자 검증 게이트로, "미래 교육 재설정"은 교수자-AI 협업 인터페이스로 구현합니다.
