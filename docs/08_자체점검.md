### H-1. CAM 용어 — Cognitive Apprenticeship Model로 단일 정의 ✅ (2026-05-17 갱신)
- **최종 결정**: `docs/cam.md` 신규 명세서 도입에 따라 **CAM = Cognitive Apprenticeship Model** (Collins/Brown/Newman 1989) 단일 정의로 확정.
- **이전 결정 폐기**: "Cognitive Activation Mechanism" 표기는 폐지.
- **이론적 보강**: Donald Schön의 **Reflection-in-Action**을 보완 이론으로 결합. CAM(시범→모방→코칭→페이딩) + Schön(작업 도중 가설·피드백 순환).
- **구조 변경**:
  - 이전 3-Phase Loop (Articulating → Coaching → Scaffolding → Modeling → Reflecting) → **2-Stage Cycle** (Articulation → Reflection)
  - 이전 6 methods 차용 → **7 Guidelines (G1~G7)** + G6 ★ CORE (Verbalize → Ground → Pass)
  - 추가: Mentor/Vending Mode 분기 + Pre-Output Self-Check (C1~C7 Hard + S1~S5 Soft)
- **공식 정의 문구**:
  > "본 시스템의 CAM은 Collins/Brown/Newman(1989)의 인지적 도제 모델(Cognitive Apprenticeship Model)이다. 본 시스템의 CAM 기반 멘토링 LLM은 CAM에 Donald Schön의 Reflection-in-Action을 결합하여 2-Stage Cycle(Articulation → Reflection)과 7 Guidelines로 구현된다."
- **학회 anchor 정합성**: 본 변경은 초록의 모듈 ③(AI-TPACK HITL) 구현 단위 보강이며 4 모듈·5 컴포넌트·진단→개입→보정 순환을 위배하지 않는다.
- **영향 받는 파일**: `06_포스터_기획설계.md`, `07_기술구현_상세.md`, `00_미디어전_마스터.md`, `09_측정변인.md`, `12_발표스크립트.md`, `13_데모_큐카드.md`, `14_데모영상_제작가이드.md`, `RE_Boot_demo_3min.srt`, `scripts/demo-video/record-with-subs.mjs`, `web/src/app/page.tsx`, `web/src/app/tutor/page.tsx`, `web/src/app/curriculum/page.tsx`

  - 학회 발표·논문: 초록 표현 그대로 ("Human-in-the-Loop 적응형 학습 플랫폼")
  - 미디어전 부스·홈·종합 정리: 보강 메시지 추가 가능 ("교수자 편차와 학습자 편차 동시 보완")
- **이유**: 학회 anchor는 변경 불가. 부스는 관람객 흡인력 위해 보강 메시지 노출 OK. 위배 아님.
- **Q&A 답변**: "이 두 메시지는 같은 건가요?" → "학회 초록의 'Human-in-the-Loop 적응형 학습 플랫폼'이 공식 표현이고, '교수자 편차와 학습자 편차 동시 보완'은 그 운영 효과를 풀어쓴 보강 메시지입니다."
- **조치 상태**: ✅ 분기 정책 명시 완료.

### L-4. 하위 25% 가드의 윤리적 정당화 보강 ✅
- **권장 조치**: 가드 ON/OFF를 학습자 본인이 선택할 수 있게 하기.
  - 기본값: 가드 ON (자기효능감 보호)
  - 마이페이지 옵션: "동기 평균 비교 정보 항상 보기" 토글 → 가드 OFF 가능
- **이유**: 가드 자체는 자기효능감 보호 목적이나, 정보 비대칭 자체가 학습자 자율성 침해라는 비판 가능. 본인 선택권 부여로 정당화.
- **Q&A 답변**: "하위 25% 가드는 학습자에게 '분류된 느낌'을 주지 않나요?" → "학습자에게 어느 그룹인지 라벨을 노출하지 않습니다. 비교 정보 표시 자체를 본인이 켜고 끌 수 있게 마이페이지에 옵션을 둡니다. 기본값은 보호 ON이지만 자기 결정권은 본인에게 있습니다."
- **조치 상태**: ✅ 정책 명시 완료. 마이페이지 UI 구현은 발표 후.

### L-5. 백엔드 CAM 프롬프트 구현 방침 결정 ✅
- **결정**: 발표는 **"설계 + 프론트엔드 데모"**로 진행. 백엔드 CAM 프롬프트는 학위논문 단계 구현.
- **이유**:
  - 미디어전 출품 산출물 기준: 설계 문서 + 작동 데모로 충분
  - 프론트엔드는 자기완결 스크립트 데모로 CAM 작동 시연 가능 (`/tutor`)
  - 백엔드 구현(SessionState · Phase 라우터 · Contrastive CoT)은 시간상 D-day 전 완성 어려움
- **Q&A 답변**: "백엔드가 실제 작동합니까?" → "현 단계에서 CAM 프롬프트 모듈은 설계가 완료된 상태이며, 본 발표는 설계와 프론트엔드 데모를 함께 보여드립니다. 후속 학위논문에서 백엔드 통합 + 실 학습자 대상 효과 측정을 진행합니다."
- **조치 상태**: ✅ 방침 결정 완료. 발표 자료에는 "설계 완료, 백엔드 통합 단계" 명시.
