# 14. 3분 데모 영상 제작 가이드 (무음 + 자막)

**용도**: 미디어전 부스 노트북 옆 대형 디스플레이 루프 재생 + 와이파이 장애 시 백업 + 학회·SNS 공유용 자료
**형식**: **무음 + 화면 캡처 + 자막 오버레이**
**총 길이**: **정확히 3분 (180초)**
**작성일**: 2026-05-14

> 본 영상은 **녹음 없이** 화면 캡처 + 자막만으로 구성한다.
> 부스 환경(소음·여러 부스 발표 동시 진행)에서 무음 재생이 오히려 자연스럽다.
> 자막은 짧고 강하게, 한 화면 당 3–5초 노출.

## 1. 녹화 환경 세팅 (촬영 15분 전)

### 1-1. 도구

| 도구 | 용도 | 비고 |
|---|---|---|
| **QuickTime Player** | 화면 녹화 (무음) | macOS 기본. ⌘+⇧+5 → 화면 일부 녹화 가능 |
| **CleanShot X** | 화면 녹화 + 마우스 강조 + 클릭 시각화 | 유료지만 마우스 클릭이 시각적으로 보여 데모에 적합 |
| **iMovie** | 컷 편집 + 자막 합성 + 내보내기 | macOS 기본. 자막 템플릿 제공 |
| **DaVinci Resolve** | 자막 + 화면 전환 + 색보정 (고급) | 무료. iMovie보다 강력하지만 학습 곡선 |
| **Premiere Pro / Final Cut** | 전문가용 | 본 영상 분량에는 과한 도구 |

**권장 조합**: **QuickTime로 녹화 + iMovie로 자막·컷**. 외장 마이크·녹음 환경 불필요.

### 1-2. 시스템 준비 (간소화)

- [ ] **화면 해상도**: 1920×1080 (Full HD)
- [ ] **브라우저**: Chrome 또는 Safari **시크릿 모드** (확장프로그램·북마크 비노출)
- [ ] **배율**: 110% (관람객 가독성)
- [ ] **시스템 알림 OFF**: 방해금지 모드 ON
- [ ] **dev 서버**: `npm run dev` + 모든 페이지 한 번씩 워밍업
- [ ] **녹화 영역**: 브라우저 전체 화면(F11) 또는 1920×1080 윈도우 캡처
- [ ] **마우스 커서 강조**: macOS 설정 → 손쉬운 사용 → 디스플레이 → 포인터 크기 키우기 (또는 CleanShot 사용)
- [ ] **테스트 녹화**: 5초 시험 녹화 → 화면 매끄러움 확인

### 1-3. 화면 캡처만 — 음성 트랙 없음

QuickTime 녹화 설정에서 **마이크 입력은 "없음"**으로 설정한다. 깨끗한 무음 영상 트랙 1개만 생성된다.

## 4. 자막 디자인 가이드

### 4-1. 읽기 속도

- 한국어 자막 권장 노출 시간: **한 줄 14–20자 = 약 3–4초**
- 두 줄(28–40자) = 약 5–6초
- 빠르게 지나가면 관람객이 못 읽음. 위 스크립트는 모두 안전 범위 내 검증 완료.

### 4-2. 폰트·크기·색

| 위치 | 폰트 | 크기 (Full HD 기준) | 색 |
|---|---|---|---|
| 중앙 (타이틀·클로징) | Pretendard Black | 96pt | 흰색 + 검은 그림자 또는 indigo 600 |
| 하단 (본문 자막) | Pretendard Bold | 48pt | 흰색 + 검은색 외곽선 2pt |
| 강조 단어 (% · 라벨) | Pretendard Black | 본문 +20% | indigo 500 또는 violet 500 |
| 출처 (모서리) | Pretendard Regular | 24pt | 흰색 70% 투명 |

### 4-3. 위치 가이드

- 하단 자막은 영상 하단으로부터 **80px 위**에 배치 (safe area)
- 중앙 자막은 영상 정중앙 — 화면 다른 요소와 겹치지 않게
- 모서리 출처는 우하단 또는 좌상단

### 4-4. 강조 처리

- **숫자**: 색·크기로 강조 — `21%`, `2,000만 원`, `1·3·7·16일`
- **이론명**: 따옴표 또는 색 — `Vygotsky ZPD`, `Bloom + Ebbinghaus`
- **CAM 라벨**: `[Articulating]`, `[Coaching]`, `[Scaffolding-Hint]` — 대괄호 유지 + 색 indigo
- **핵심 한 줄**: 마지막 클로징 `AI가 분석하고, 교수자가 결정합니다.`는 크게, 1.5초 페이드인

### 4-5. 전환 효과

- 자막 사이 전환: 페이드 인/아웃 0.3초
- 화면 전환(페이지 이동): 부드러운 cut 또는 0.5초 디졸브
- 화면 캡처 자체는 가속 금지 (1배속 유지) — 단, 폼 스크롤 같은 느린 구간만 1.5배

## 5. 편집·내보내기

### 5-1. iMovie 워크플로우 (권장)

1. QuickTime로 녹화된 mov 파일을 iMovie에 임포트
2. 타임라인에 배치 → 위 스크립트 따라 컷 편집 (불필요한 구간 잘라내기)
3. 각 구간에 **자막 (Titles → Lower Third 또는 Centered)** 추가
4. 자막 텍스트는 위 2절 스크립트 그대로 입력
5. 자막 시작·종료 시간 조정 (각 자막 3–5초 노출 보장)
6. 필요 시 **BGM 추가** (선택사항, 5-3 참조)
7. 최종 미리보기 후 내보내기

### 5-2. 자막 합성 옵션

| 옵션 | 장점 | 단점 |
|---|---|---|
| **Burned-in 자막** (영상에 합성) | 어디서나 동일하게 보임 | 수정 시 영상 재출력 필요 |
| **SRT 별도 파일** | 켜고 끌 수 있음 | 플레이어 의존 |
| **둘 다 제공** | 부스 재생용 burned-in + 공유용 SRT | 작업 2배 |

**권장**: 부스 재생용 **burned-in** 자막 필수. SRT 파일은 YouTube 업로드 시 자동 생성되므로 선택.

### 5-3. BGM (선택 — 부스 환경 고려 권장)

부스 디스플레이가 **무음 루프 재생**되므로 BGM은 선택사항. 다만:

- **부스가 조용한 시간대** (오전 일찍, 점심시간 등): BGM 있으면 관람객 시선 유도에 도움
- **다른 부스 음성이 들리는 시간대**: 무음이 오히려 깔끔

만약 BGM을 넣는다면:
- 저작권 무료: YouTube Audio Library, Pixabay, Artlist 무료 트랙
- 톤: 잔잔한 piano / lo-fi instrumental
- 음량: -20 dB ~ -25 dB (배경)
- 발화가 없으므로 음량 충돌 걱정 없음

### 5-4. 내보내기 사양

| 항목 | 값 |
|---|---|
| 해상도 | 1920×1080 (Full HD) |
| 프레임 | 30 fps |
| 코덱 | H.264 |
| 비트레이트 | 8 Mbps |
| 형식 | MP4 |
| 오디오 트랙 | **없음** (또는 BGM만) |

| 크기 | ~100 MB 이내 (무음이라 더 가벼움) |

## 6. 활용 시나리오

### 6-1. 부스 디스플레이 루프 재생 ★ 메인 용도

## 8. 체크리스트

- [ ] 총 길이 정확히 3분 (±5초 허용)
- [ ] 모든 자막이 3–5초 노출 (너무 빨리 사라지지 않음)
- [ ] 자막이 화면 안전 영역(safe area) 내에 위치 (가장자리 80px 여백)
- [ ] CAM 튜터 시연 시 [Articulating] [Coaching] [Scaffolding-Hint] 라벨이 자막으로 정확히 표시
- [ ] 강조 단어(숫자·이론명·라벨)에 색·크기 강조 적용
- [ ] 출처 표기: 박진아·김지은(2024), Fernandes et al.(2025) 자막 또는 영상 끝에 명시
- [ ] 첫·끝 화면에 연구자명·소속 명시 (김혜진 · 연세대학교 교육공학)
- [ ] MP4 파일 크기 < 150 MB
- [ ] 오디오 트랙이 무음이거나 BGM만 있음 (육성 X)
- [ ] 노트북·외장 디스플레이에서 풀스크린 루프 재생 테스트
- [ ] 자막이 한 화면에 최대 2줄까지만 표시됨

## 9. 단축 팁 (시간 부족 시)

녹화·편집 시간을 더 줄여야 한다면:

1. **iMovie 자막 템플릿 활용** — 직접 디자인하지 말고 "Lower Third" 또는 "Centered Title" 템플릿 그대로 사용
2. **화면 캡처는 한 번에** — 페이지별로 끊지 말고 전체 데모를 한 번에 녹화한 뒤 편집에서 자르기
3. **자막은 한국어만** — 영문 자막은 학위논문·국제 학회 단계에서 추가
4. **BGM 생략** — 부스 무음 재생이면 BGM 없어도 무방
5. **CleanShot 같은 자동 클릭 시각화 도구** — 마우스 강조 효과를 후처리하지 않고 녹화 시 자동 합성
