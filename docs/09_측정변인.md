# 09. 측정 변인 — AI 과의존 방지 효과 검증

**문서 목적**: 학회 초록의 "AI 과의존을 구조적으로 방지" 주장을 측정 가능한 변인으로 변환한다.
**작성일**: 2026-05-14
**관련 anchor**: `05_제출초록.md` (모듈 ③ AI-TPACK HITL 협업)

> 초록은 "AI 과의존을 구조적으로 방지한다"고 명시한다. 학회 패널이 "그 효과를 어떻게 측정하나"를 물을 가능성이 높다. 본 문서는 본 시스템의 메커니즘(즉답 차단, CAM, Socratic, 체크리스트 증거 입력, 망각곡선 재인출)에서 직접 추출 가능한 **4 변인**과 보조 지표를 정의한다.

---

## 1. 핵심 4 변인 (Primary)

### V-1. Socratic 즉답 요구 빈도 (System-1 Demand Rate)

| 항목 | 내용 |
|---|---|
| **정의** | 한 학습 세션 내 학습자가 "빨리", "그냥 답", "정답만" 류 즉답 요구 표현을 사용한 횟수 |
| **측정** | CAM LLM 대화 로그에서 키워드·의도 분류로 추출 (Contrastive CoT 신호 추출 사용) |
| **단위** | 횟수 / 세션 |
| **방향** | **낮을수록 좋음** (AI 과의존 ↓) |
| **베이스라인** | 일반 GPT 사용 세션의 즉답 요구 빈도 |
| **결과 해석** | RE:Boot 세션 평균 < 베이스라인 → AI 과의존 방지 효과 |
| **데이터 출처** | `api/app/services/tutor/` 대화 로그 + Contrastive 분류 |

---

### V-2. 회고 응답 깊이 (Reflection Depth Score)

| 항목 | 내용 |
|---|---|
| **정의** | Phase 3(Reflecting)에서 학습자가 작성한 회고 문장의 (a) 길이, (b) 본인 사례 인용 여부, (c) 일반 원칙 연결 여부의 가중 점수 |
| **측정** | LLM 자동 평가 (Haiku) — 짧은 단답("이해함") vs 본인 사례 기반 일반화 회고 분류 |
| **단위** | 0–10 점 (단답 0, 일반화 회고 10) |
| **방향** | **높을수록 좋음** (메타인지 활성 ↑) |
| **베이스라인** | 본인의 첫 세션 회고 점수 (within-subject) |
| **결과 해석** | 세션 수 증가에 따라 점수 상승 → 메타인지 보정 효과 |
| **데이터 출처** | CAM LLM 대화 Phase 3 발화 |

---

### V-3. 응용 질문 정답률 (Transfer Accuracy)

| 항목 | 내용 |
|---|---|
| **정의** | Phase 3에서 LLM이 제시하는 응용 질문(같은 개념의 다른 맥락 적용 문제)에 대한 학습자 정답률 |
| **측정** | CAM LLM이 자동 생성하는 1개 응용 질문 + 학습자 응답 + LLM 채점 |
| **단위** | 정답률 (%) |
| **방향** | **높을수록 좋음** (실제 이해 ↑, 능력 착각 ↓) |
| **베이스라인** | 동일 세션의 학습자 자기보고("이해했다") 비율 |
| **결과 해석** | 자기보고 "이해했다" 비율 대비 응용 정답률의 격차가 작을수록 능력 착각 적음 |
| **데이터 출처** | CAM LLM 대화 Phase 3 응용 질문 결과 |

---

### V-4. 대화 신호 ↔ 체크리스트 증거 입력 일치도 (Coherence Index)

| 항목 | 내용 |
|---|---|
| **정의** | CAM 대화에서 학습자가 "이해함"으로 발화한 개념과, 수업 후 체크리스트의 해당 항목 증거 입력 품질의 일치 여부 |
| **측정** | (1) 대화에서 자기보고 "이해함" 발화한 개념 추출 → (2) 체크리스트의 해당 항목 증거 입력 길이·키워드 정합도 평가 |
| **단위** | 0–1 (1: 자기보고와 증거 입력 일치, 0: 자기보고만 있고 증거 입력 부재·부실) |
| **방향** | **높을수록 좋음** (메타인지 보정 ↑) |
| **베이스라인** | 본인의 누적 평균 |
| **결과 해석** | 일치도 < 0.5 → 능력 착각 신호 (강사 대시보드 알림 트리거) |
| **데이터 출처** | CAM 대화 로그 + 체크리스트 증거 입력 |

## 2. 보조 지표 (Secondary)

| 지표 | 정의 | 사용처 |
|---|---|---|
| **사전·사후 자기평가 격차** | 학습목표 자기평가에서 사전 "전혀 모름" → 사후 "할 수 있음"으로 뛴 비율 | 능력 착각 의심 신호 (premature_confidence) |
| **체크리스트 증거 미입력 비율** | 체크 표시는 했으나 증거 칸이 비어있는 항목 비율 | 형식적 통과 신호 |
| **망각곡선 D+1 정답률** | 1일 후 복습 퀴즈 정답률 | 지식 정착 수준 |
| **CAM 응답 범위 외 질문 빈도** | 학습자가 강사 설정 범위 밖 질문을 시도한 빈도 | 관심 확장 vs 회피 분류 |

---

## 3. Risk Score 가중치 종합 (강사 대시보드)

학습자별 Risk Score는 위 변인의 가중합으로 산출된다. 가중치 합 = 100.

| 컴포넌트 | 가중치 | 근거 변인 |
|---|---|---|
| 0) 전체 과정 이수율 (동기 평균 대비 위치) | 20 | 사회비교 + 사전·사후 격차 |
| 1) 학습목표 자기평가 (사전·사후 격차) | 15 | V-2 회고 깊이 (간접) |
| 2) 체크리스트 (증거 미입력 비율) | 15 | V-4 일치도 |
| 3) CAM LLM (즉답 요구 + 얕은 회고) | 10 + 10 | V-1 즉답 요구 + V-2 회고 깊이 |
| 4) 망각곡선 퀴즈 (정답률 50% 미만) | 20 | V-3 응용 정답률 (간접) |
| 가산: deep_engagement 발견 | -10 | V-2 회고 깊이 ≥ 7 |
| **합계** | **100 (최대)** | |

Risk Score ≥ 60 → 강사 대시보드 "지원 필요" 분류
40 ≤ Score < 60 → "관찰"
Score < 40 → "안정"

## 4. 측정 가능성 / 미구현 표시

| 변인 | 현재 구현 상태 | D-day까지 가능성 |
|---|---|---|
| V-1 즉답 요구 빈도 | 코드 안에 키워드 매칭 기본 골격 존재(`select_mode` in `07_기술구현_상세.md` 7-6) | 가능 |
| V-2 회고 깊이 | 미구현 (Contrastive 프롬프트 설계는 있음, 8-2) | 부분 가능 (스코어 함수 추가) |
| V-3 응용 정답률 | 미구현 (Phase 3 프롬프트는 있음, 6-5) | 부분 가능 |
| V-4 일치도 | 미구현 (대화 ↔ 체크리스트 조인 로직 필요) | 발표 시점에는 개념만 제시 |

**발표 전략**: V-1·V-2는 코드에 부분 구현되어 있어 라이브 데모 가능. V-3·V-4는 설계 문서·로직 흐름으로 제시. 학회 미디어전은 "설계 + 구현 + 측정 변인 정의"까지를 산출물로 인정한다는 점을 활용.
