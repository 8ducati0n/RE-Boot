# Part 0. 제출 초록 (공식 기록)

> 2026 한국교육정보미디어학회 춘계학술대회 발표신청서에 제출된 초록 원문이다. 후속 모든 문서는 이 초록과 충돌하지 않도록 작성한다.

## 제목
**부트캠프형 SW·AI 교육의 중도 이탈률 감소와 AI 과의존 방지를 위한 Human-in-the-Loop 적응형 학습 플랫폼 'RE:Boot' 설계 및 개발**

## 초록 본문
국내 부트캠프형 SW·AI 교육의 중도 이탈률은 약 21%에 이르며(박진아·김지은, 2024), 이에 따른 1인당 국비 손실은 최대 2,000만 원에 달한다. 동시에 AI 기반 교육 도구의 확산과 함께 학습자의 AI 과의존 문제가 주요 리스크로 부각되고 있다(Fernandes et al., 2025). 본 연구는 이 두 문제를 동시에 다루기 위해 AI의 확장성과 교수자 판단의 신뢰성을 결합한 Human-in-the-Loop 적응형 학습 플랫폼 'RE:Boot'를 설계·구현하였다. RE:Boot는 다음 네 가지 핵심 모듈을 **'진단 → 개입 → 보정'**의 순환 구조로 통합한다.

1. **첫째**, Vygotsky의 근접발달영역(ZPD) 이론에 기반한 수준 진단 및 갭 맵을 통해 학습자 간 사전지식 편차를 정량화한다.
2. **둘째**, Bloom의 완전학습 모형과 Ebbinghaus 망각곡선을 접목한 형성평가 및 간격 반복 시스템을 통해 지식 정착을 지원한다.
3. **셋째**, AI-TPACK 프레임워크에 따라 AI가 교육적 개입을 제안하고 교수자가 이를 승인하는 협업 모델을 적용하여, AI 과의존을 구조적으로 방지한다.
4. **넷째**, 학습분석 기반 조기경보 시스템과 데이터 기반 동료 학습 그룹핑을 통해 이탈 위험 학습자를 선제적으로 지원한다.

본 플랫폼은 AI의 모든 교육적 제안이 교수자 승인을 거쳐야만 학습자에게 전달되는 구조를 채택함으로써, 확장 가능하면서도 신뢰할 수 있는 AI 교육 환경을 실현하고자 한다.

## A-1. 한 줄 정의

**RE:Boot = 부트캠프 중도 이탈과 AI 과의존을 동시에 해결하는 Human-in-the-Loop 적응형 학습 플랫폼**

학습자에게는 메타인지 착각 완화와 학습 지속성을, 교수자에게는 데이터 기반 학습자 이해와 정서 지원 의사결정을, 운영사·사회에는 이탈률 감소와 일자리 순환 안정성을 제공한다.

| 문제 | 현상 | 근거 |
|---|---|---|

| # | 모듈 | 이론 기반 | 작동 방식 |
|---|---|---|---|

## A-6. 시스템 역할 구조 (HITL)

```
   AI                          교수자(인간)
   ──                          ─────────
   · 지식 전달                   · AI 산출물 선택·검증 (HITL)
   · 학습 데이터 수집·분석        · 학습자 정서적 지원
   · Socratic 응답 (직답 회피)    · 개입 의사결정
                  ↓                  ↑
              ┌──── 학습자 ────┐
              │ · 자기 위치 인식 │
              │ · 학습 지속      │
              └─────────────────┘
```

- AI의 **모든** 교육적 제안은 교수자 승인을 거쳐야만 학습자에게 전달된다.
- 정서적 위로·완성 메시지 생성은 AI 금지 영역이다.

## A-7. 메타인지 착각 완화 4장치

| 시점 | 장치 | 작동 주체 | 핵심 원리 |
|---|---|---|---|
| **수업 전** | 체크리스트 / 학습목표 / 전체 과정 이수율 점검 | 학습자 (컴포넌트 0·1·2) | 자기 위치·도달 수준의 메타적 인식 |
| **수업 중 (질문 응답)** | CAM 기반 멘토링 LLM — 2-Stage Cycle | 학습자 (컴포넌트 3) | Articulation(명료화) → Reflection(성찰) 순환으로 즉답 차단 + 사고 외현화 |
| **수업 중 (인지 활성)** | CAM 7 Guidelines — G6 ★ Verbalize → Ground → Pass | 학습자 (컴포넌트 3) | 업로드물에 즉답·평가 금지. 3-Step으로 환각 차단 + 능력 착각 차단 |
| **수업 외** | 교수자 정서 개입 (데이터 기반) | **강사 (대시보드)** | 인지적 어려움 구간에서 강사가 직접 개입 |

> **CAM = Cognitive Apprenticeship Model** (Collins/Brown/Newman 1989). 본 시스템의 멘토링 LLM은 CAM에 Donald Schön의 Reflection-in-Action을 결합해 2-Stage Cycle과 7 Guidelines로 구현한다. 상세는 `07_기술구현_상세.md` §5.

→ **4장치 ↔ 5 컴포넌트 대칭성 주의**: 4장치 중 셋(수업 전·수업 중 질문·수업 중 인지 활성)은 학습자 5 컴포넌트로 직접 구현되지만, **"수업 외" 장치는 학습자 컴포넌트가 아니라 강사 측 메커니즘**이다. 본 분리는 v2.1-C의 "AI는 사실만 정리, 해석·결정은 강사" 원칙과 B-8-4(모듈 ④ 강사 측 처리)와 정합한다.

| 초록의 4 모듈 | 학습자 5 컴포넌트 | 메타인지 4장치 | 핵심 이론 |
|---|---|---|---|
| ① 수준 진단 + 갭 맵 | 0) 전체 과정 이수율 · 1) 학습목표 (수업 전) | 수업 전 | ZPD |
| ② 형성평가 + 간격 반복 | 2) 체크리스트 (수업 후) · 4) 망각곡선 퀴즈 | — (지식 정착) | Bloom + Ebbinghaus |
| ③ AI-TPACK HITL 협업 | 3) Socratic CAM LLM + 강사 F1 등록·승인 | 수업 중 (Socratic + CAM) | AI-TPACK |
| ④ 조기경보 + 동료 그룹핑 | (강사 대시보드 + 동료 그룹핑 기능) | 수업 외 (정서 개입) | Learning Analytics |

→ 초록의 4 모듈은 **이론적 frame**, 학습자 5 컴포넌트는 **구현 단위**, 메타인지 4장치는 **메커니즘 단위**다. 세 축이 같은 시스템의 다른 단면이다.

## B-2. 시스템 역할 구조

### AI의 역할
- 지식 전달
- 학습자 행동·학습 데이터 수집 및 분석
- Socratic 방식 질문 응답 (직답 회피)

### 교수자(인간)의 역할
- AI 산출물의 선택과 검증 (HITL)
- 학습자에 대한 정서적 지원
- 분석 데이터 기반 개입 의사결정

### 학습자 대상 효과
- 메타인지 착각 완화
- 학습 지속성 유지

### B-2-보강. "AI 산출물의 선택과 검증"의 확장성 — 자동 통과 / 수동 검토 분리

초록은 본 플랫폼을 "확장 가능하면서도 신뢰할 수 있는" 환경으로 명시한다. 교수자가 24명 × 매일 × 다수의 AI 제안을 모두 수동 검토하면 확장성이 무너지므로, **AI 산출물은 두 갈래로 분리되어 흐른다**.

| 흐름 | 자동 통과 (운영 자동화) | 수동 검토 (강사 결정) |
|---|---|---|
| **대상** | 정해진 스케줄·규칙으로 발송되는 산출물 | 학습자에게 가는 메시지·해석·결정 |
| **예시** | · 망각곡선 퀴즈 스케줄링·발송<br>· 학습목표 자기평가 노출<br>· 체크리스트 자동 큐잉<br>· grounding score 높은 정형 추천 | · 학습자에게 보낼 메시지 작성·발송<br>· 동료 그룹핑 결정<br>· 재학습 자료 큐레이션<br>· 위험 신호 개입 시점·방식 결정 |
| **책임 귀속** | 운영 절차 (강사가 사전 설정) | 강사 본인 |
| **메시지 생성** | 강사가 사전 정의한 템플릿 | 강사가 직접 작성 (Composition Assistant는 사실·옵션·표현 재료만 제공) |

**원칙**: 학습자에게 가는 **모든 정서적·해석적 발화**는 강사 검토를 거친다. 운영 절차는 자동화하되, 학습자 관계에 영향을 주는 결정은 사람이 한다. 이 분리가 초록의 "확장성"과 "교수자 판단의 신뢰성"이 양립하는 근거다.

## B-3. 메타인지 착각 완화 메커니즘

### B-3-1. 수업 전 (사전 점검 단계)
- 체크리스트 작성
- 학습목표 확인
- 전체 과정 이수율 점검
- → 학습자가 자신의 위치와 도달 수준을 메타적으로 인식하게 함
- → 초록의 모듈 ① (ZPD 진단·갭 맵)과 정합

### B-3-2. 수업 중 (질문·응답 개입)
- **CAM (Cognitive Apprenticeship Model) 기반 멘토링 LLM 적용**: Collins/Brown/Newman(1989)의 인지적 도제 모델 + Donald Schön의 Reflection-in-Action을 결합. 학습자의 사고를 외현화(externalize)하도록 유도하는 코치 역할.
- **2-Stage Cycle**: 모든 응답은 **Articulation(명료화) → Reflection(성찰)** 순환으로 작동. 즉답·평가 금지.
- **7 Guidelines (G1~G7)**: 응답 생성 전 7개 가이드라인 통과 필수. **G6 ★ CORE**는 업로드물(코드·디자인·데이터)에 대한 3-Step 절차 — *Verbalize(언어화) → Ground Truth(확인) → Pass(질문 패스)*.
- **출력 자체 검증**: 응답 전송 직전 C1~C7(Hard) + S1~S5(Soft) 체크리스트 통과해야 함.
- → 초록의 모듈 ③ (AI-TPACK HITL)의 학습자-측 작동 방식

**G6 절대 금지**
- 업로드 직후 "이 부분은 개선이 필요합니다", "이렇게 바꾸세요" 등 평가·조언을 먼저 출력하는 것
- Step 1(Verbalize) 없이 Step 3(Pass)로 건너뛰는 것
- Step 2(Ground Truth 확인) 없이 분석을 진행하는 것

**Mentor Mode / Vending Mode 분기**
- **Mentor Mode** (기본): 7 Guidelines 전체 활성, 사고 확장
- **Vending Mode** (학습자 요청 시): 즉답 허용. 강사 대시보드에 모드 전환 사실 통보
- 상세는 `07_기술구현_상세.md` §5 참조.

### B-3-3. 수업 외 (교수자 정서 개입)
- 학습자의 인지적 귀찮음·어려움 구간에서
- 교수자가 AI가 분석한 학습자 데이터를 실시간으로 확인
- 정서적 support 제공
- → 메타인지 착각 완화 + 학습 지속성 유지 동시 달성
- → 초록의 모듈 ④ (조기경보 + 동료 그룹핑) + 교수자 정서 개입

### B-3-보강. ZPD 통합 구현 — 갭 맵 + Socratic CAM + 강사 면담

Vygotsky의 ZPD는 단순한 진단 도구가 아니라, **혼자서는 도달할 수 없는 영역을 보다 능숙한 타자(MKO: More Knowledgeable Other)의 중재로 도달하게 하는 사회적 학습 구조**다. 본 시스템은 ZPD의 세 구성 요소를 다음과 같이 분담 구현한다.

| ZPD 구성 요소 | 본 시스템 구현 | 학습자 컴포넌트 |
|---|---|---|
| **현재 발달 수준 측정** | 수준 진단 + 갭 맵 + 학습목표 자기평가 | 0) 이수율 · 1) 학습목표 |
| **잠재 발달 수준으로의 중재 (Scaffolding)** | Socratic CAM LLM의 graduated sequence (Articulation → Coaching → Scaffolding → Modeling) | 3) Socratic CAM LLM |
| **사회적 상호작용 (MKO)** | 교수자 면담 · 동료 그룹핑 (Tier 2 강사 결정) | (강사 측 — B-8-4 참조) |

→ 갭 맵 단독은 진단 도구일 뿐이며, **갭 맵 + CAM 중재 + 강사 면담**의 결합이 ZPD의 실제 구현이다. 학회 질의응답 시 "ZPD를 갭 맵 하나로 정당화할 수 있나요"에 대한 답변 근거.

## B-6. 시장 필요성 (Why Now / Why This)

### B-6-1. 데이터 인프라 공백
- LMS를 보유한 교육 운영사가 있는 반면, 미보유 운영사도 존재
- LMS 미보유 운영사는 학습자 데이터 축적·분석 기반이 부재
- → RE:Boot 같은 플랫폼 서비스가 필수적

### B-6-2. 서울시 운영 부트캠프 현장의 구조적 편차

**교육운영사 단위 편차**
- 운영사에 따라 교육 운영방식의 편차가 큼
- 동일 사업 내에서도 학습자 경험이 균질하지 않음

**강사(교수자) 단위 편차**
- 강사는 도메인 스킬·지식은 보유
- 그러나 그 외 교수자로서 갖춰야 할 역량에서는 편차가 큼
- 특히 SW 중심 교과 특성상 비도메인 교수 역량(학습자 이해, 정서적 지원, 동기 부여 등)이 구조적으로 부족

**지원 체계의 한계**
- 교육운영사와 서울시 산하 기관에서 취업 관련 서포트는 존재
- 그러나 학습자와 가장 오랜 시간을 보내는 주체는 강사
- 따라서 강사의 학습자 이해와 적극적 지원이 결과에 결정적
- 현재 구조에서는 이 부분이 충족되지 못함

## B-8. 최종 기획 — 강사 입력 + 학습자 5 컴포넌트

> 이 절의 구조는 변경되지 않는 최종 기획이다. 모든 후속 개발·디자인·논문 작업은 이 7-요소를 단위로 진행한다.

### B-8-1. 강사 측 — 커리큘럼 등록 칸 (F1)

강사가 다음을 입력하면, 학습자 측 5 컴포넌트가 자동 생성된다.

| 입력 필드 | 비고 |
|---|---|
| 주차 / 회차 / 주제 | 학습자 컴포넌트 0(이수율) 산출 기준 |
| 학습목표 (행동 동사 기반) | 학습자 컴포넌트 1(학습목표)로 흐름 |
| 사전 요구 지식 | 모듈 ① 갭 맵 진단 보강 큐 |
| 수업 후 체크리스트 (증거 입력형) | 학습자 컴포넌트 2(체크리스트)로 흐름 |
| CAM LLM 응답 범위 | 학습자 컴포넌트 3(Socratic CAM LLM) 가드 |
| 퀴즈 풀 | 학습자 컴포넌트 4(망각곡선 퀴즈)로 흐름 |
| 망각곡선 간격 | 동(同) |

### B-8-2. 학습자 측 — 커리큘럼 바탕으로 자동 생성되는 5 컴포넌트

**0) 전체 과정 이수율 점검** — 항상 확인
- 학습자가 부트캠프 전 기간 동안 자기 위치를 메타적으로 인식한다.
- 메타인지 4장치의 "수업 전" 인식 축에 해당한다.
- 초록 모듈 ① 매핑.

**1) 학습목표** — 수업 전 확인
- 수업 시작 전 오늘 학습할 목표를 자기평가한다.
- 메타인지 4장치의 "수업 전" 인식 축에 해당한다.
- 초록 모듈 ① 매핑.

**2) 체크리스트** — 수업 후 확인
- 수업 직후 학습한 내용을 자기점검한다.
- "안다는 느낌"이 아닌 "실제 수행"을 분리하기 위해 증거 입력형으로 작동한다.
- 초록 모듈 ② 매핑 (Bloom 완전학습).

**보강. 증거 입력의 실효성 가드** (학습자가 의미 없는 한 줄로 통과하는 것 방지)
| 가드 | 작동 |
|---|---|
| **본인 회기 재검토** | 다음 회기 시작 시 본인이 직전 회기에 작성한 증거 입력을 다시 본인이 검토하게 한다. "이게 진짜 그 학습목표를 보여주는가" 자기 확인 단계 추가. |
| **최소 키워드 가드** | 강사가 F1에서 설정한 학습목표·CAM 응답 범위 키워드와의 정합도를 검사. 일치 0 → 강사 대시보드에 "증거 부실 신호" 표시 (학습자 노출 없음, 강사만). |
| **AI 자동 채점은 의도적으로 배제** | AI 검증을 추가하면 다시 AI 의존이므로 의도적으로 배제. 대신 본인 재검토 + 강사 확인의 2단계 가드만 적용. |

→ **"증거 입력이 의미 없는 한 줄이면 어떻게 하나요"** 질문 대비: 본인 재검토 단계와 강사 측 키워드 가드로 형식적 통과를 차단한다. AI 검증은 의도적으로 배제한다.

**3) Socratic CAM LLM** — 실습 중 · 자습 중 확인
- 코딩 실습 또는 자습 중 질문이 발생할 때 호출된다.
- Socratic 응답으로 즉답을 차단하고, CAM으로 능력 착각을 차단한다.
- 메타인지 4장치의 "수업 중" 축에 해당한다.
- 초록 모듈 ③ 매핑 (AI-TPACK HITL).

**4) 퀴즈 (에빙하우스 망각곡선 pop-up)** — 수업 후 며칠 뒤
- 수업 후 정해진 간격(예: 1·3·7·16일)에 따라 pop-up으로 노출된다.
- 장기 기억 강화 + 재인출을 통한 메타인지 보정.
- 초록 모듈 ② 매핑 (Ebbinghaus 망각곡선).

**보강. 망각곡선 적용의 도메인 적합성**
Ebbinghaus 원전(1885)은 무의미 음절 암기 연구이며, 코딩 학습은 declarative 지식보다 procedural 지식 비중이 크다. 본 시스템은 이를 다음과 같이 보완한다.

| 보완 요소 | 적용 |
|---|---|
| **적응형 간격 (adaptive spacing)** | 1·3·7·16일은 초기값일 뿐. 정답률 80%↑ → 다음 간격 연장(16일 → 30일), 50%↓ → 단축. SuperMemo SM-2 / Anki와 동일한 원리. |
| **재인출 효과 (Testing Effect)** | 단순 노출이 아닌 코드 빈칸·단답·객관식 혼합 퀴즈로 능동적 재인출 강제. Karpicke & Roediger(2008)의 testing effect 적용. |
| **분산 연습 (Distributed Practice)** | Bloom 완전학습 형성평가 + 망각곡선 재인출을 한 사이클에 결합. Cepeda et al.(2006) meta-analysis 정합. |
| **코딩 도메인 spaced practice** | 개념 회상뿐 아니라 짧은 코드 작성·실행 결과 예측 문항 포함. procedural 지식 retention 보강. |

→ **"Ebbinghaus 일률 적용 아닌가요"** 질문 대비: 본 시스템은 Ebbinghaus를 출발점으로 하되 적응형 spacing + testing effect + 분산 연습 + 코딩 도메인 문항으로 보완한다.

### B-8-3. F1 ↔ 5 컴포넌트 매핑

```
[강사 입력 F1]                  [학습자 측 자동 생성]
─────────────────────         ─────────────────────────
주차/회차/주제                 →  0) 전체 과정 이수율
학습목표                       →  1) 학습목표 (수업 전)
체크리스트(증거 입력형)         →  2) 체크리스트 (수업 후)
CAM 응답 범위                  →  3) Socratic CAM LLM
퀴즈 풀 + 망각곡선 간격         →  4) 망각곡선 퀴즈 pop-up
```

### B-8-4. 모듈 ④ (조기경보 + 동료 그룹핑)의 학습자 측 컴포넌트 부재 사유

초록의 모듈 ④는 **강사 측 메커니즘**으로 작동한다. 학습자 측에 별도 컴포넌트가 없는 것은 의도된 설계다.

| 항목 | 처리 위치 | 이유 |
|---|---|---|
| **조기경보** | 강사 대시보드의 사실 패널 + Risk Score | 위험 신호의 해석·개입 결정은 교수자의 책임 영역(HITL). 학습자에게 "당신은 위험군"이라는 라벨을 노출하지 않아 자기효능감을 보호한다. |
| **동료 그룹핑 결정** | 강사 대시보드 (Tier 2 승인 필수) | 그룹 구성은 학습자 데이터를 통합 판단해야 하므로 교수자 의사결정 사안이다. 학습자에게는 **결정된 결과(그룹 매칭 통보)**만 통보된다. |
| **그룹 활동 진행** | 별도 협업 채널 (Slack·Discord·Notion 등 운영사 기존 도구) | 본 플랫폼은 그룹 결정·메시지에 집중하고, 그룹 내 협업 자체는 기존 도구를 신뢰한다. |

**원칙**: 학습자 5 컴포넌트는 메타인지 보정과 자기점검에 한정하고, 조기경보·그룹핑은 교수자 책임 귀속을 명확히 한다. 본 분리는 v2.1-C의 핵심 원칙인 "AI는 사실만 정리, 해석·결정은 강사"와 정합한다.
