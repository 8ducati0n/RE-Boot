# 1. 출품 메시지 (Hero)

## 한 줄 정의

> **AI가 분석하고, 교수자가 결정합니다.**
> 부트캠프 중도 이탈률 21%와 AI 과의존을 동시에 해결하는 Human-in-the-Loop 적응형 학습 플랫폼.

## 학회 제출 제목 (변경 불가)

**부트캠프형 SW·AI 교육의 중도 이탈률 감소와 AI 과의존 방지를 위한 Human-in-the-Loop 적응형 학습 플랫폼 'RE:Boot' 설계 및 개발**

## 핵심 메시지 3가지

1. **두 문제는 한 뿌리**: 부트캠프 중도 이탈도, AI 과의존도, 모두 **메타인지 실패**의 두 얼굴이다.
2. **'진단 → 개입 → 보정' 한 사이클**: ZPD 진단 → AI-TPACK HITL 개입 → Bloom·Ebbinghaus 보정.
3. **AI는 인지, 교수자는 정서**: 역할을 나누는 것이 곧 학습자 보호 장치다. AI의 모든 제안은 교수자 승인을 거친다.

적응형 학습 · Human-in-the-Loop · 부트캠프 중도 이탈 · AI 과의존 · 학습분석

## 3-2. Q&A 핫토픽 (예상 질문)

| 질문 | 핵심 답변 |
|---|---|
| "Socratic AI가 학습자에게 답답하지 않나요?" | 마감 임박·동일 개념 3턴 이상 막힘 시 graduated sequence(Hint→Principle→Knowledge→Modeling)로 자동 완화. |
| "Ebbinghaus 망각곡선을 일률 적용한 것 아닌가요?" | 출발점일 뿐. 정답률 기반 **적응형 간격**(SuperMemo SM-2 / Anki 원리) + **재인출 효과**(Karpicke & Roediger 2008) + **분산 연습**(Cepeda et al. 2006) + **코딩 도메인 procedural 문항**(코드 빈칸·실행 결과 예측)으로 보완합니다. |
| "체크리스트 증거 입력이 형식적이면 어떻게 하나요?" | 본인 회기 재검토 + 강사 측 키워드 가드의 **2단계 가드**로 차단합니다. AI 자동 검증은 의도적으로 배제합니다 (AI 의존 회피). |
| "4장치 중 '수업 외'는 왜 학습자 컴포넌트가 없나요?" | 의도된 설계입니다. 수업 외 정서 개입은 강사 측 메커니즘으로 작동하며, 학습자에게는 결과(강사 메시지)만 전달됩니다. v2.1-C의 "AI는 사실만 정리, 해석·결정은 강사" 원칙과 정합합니다. |
| "교수자가 모든 메시지를 직접 쓰는 게 확장 가능한가요?" | Tier 1 자동 통과(퀴즈 스케줄·체크리스트 발송 등)와 Tier 2 강사 승인(메시지·그룹핑)으로 분리. |
| "이게 LMS와 뭐가 다른가요?" | LMS는 자료 전달·출결 중심. RE:Boot는 메타인지 보정과 정서 개입 의사결정 보조 중심. LMS 위·아래·옆 어디에든 들어갈 수 있다. |
| "AI 과의존을 어떻게 측정하나요?" | (1) Socratic 즉답 요구 빈도, (2) 회고 깊이, (3) 응용 질문 정답률, (4) CAM 대화 ↔ 체크리스트 증거 일치도 네 변인. |
| "AI-TPACK이 교수자를 단순 검수자로 만들지 않나요?" | 대시보드에 AI의 추론 근거를 노출해 교수자가 학습한 뒤 의사결정. 단순 승인 버튼이 아님. |
| "CAM은 어디서 온 개념인가요?" | **CAM = Cognitive Apprenticeship Model** (Collins/Brown/Newman 1989). 인지적 도제 모델로, 장인-제자의 시범→모방→코칭→페이딩을 추상적 인지 과정에 적용한 학습 모델입니다. 본 시스템은 여기에 **Donald Schön의 Reflection-in-Action**을 결합해 **2-Stage Cycle**(Articulation→Reflection)과 **7 Guidelines**로 구현했습니다. 상세 명세는 `docs/cam.md`. |
| "Mentor/Vending 모드는 뭔가요?" | 사용자가 사고 확장을 원할 때는 Mentor Mode(7 Guidelines 전체 활성, 즉답 차단), 빠른 결과가 필요할 때는 Vending Mode(즉답 허용). 학습자가 명시 요청하거나 강사가 사전 설정으로 전환합니다. 메타인지 자각의 핵심 — 학습자가 자기 모드를 의식적으로 선택하게 합니다. |
| "G6 3-Step이 핵심이라는데 뭐가 다른가요?" | 학습자가 코드·디자인·자료를 올리면 AI는 무조건 ① **Verbalize**(제가 본 것은 ~입니다) → ② **Ground Truth 확인**(제가 잘못 본 부분이 있을까요?) → ③ **Pass**(질문으로 종료) 3-Step을 거칩니다. 환각 차단 + 즉답 차단 + 사고 외현화를 한 절차로 달성합니다. |
| "ZPD를 갭 맵 하나로 정당화할 수 있나요?" | 갭 맵은 진단 도구일 뿐. **갭 맵(현재 발달 수준 측정) + Socratic CAM graduated sequence(Scaffolding) + 강사 면담·동료 그룹핑(MKO 사회적 상호작용)**의 결합이 ZPD의 실제 구현. (`06_포스터_기획설계.md` B-3-보강) |
| "Socratic이랑 CAM이랑 같은 거 아닌가요?" | 직교 개념. **Socratic = 응답 패턴**(역질문 양식). **CAM 구현 방법론(Cognitive Apprenticeship Model) = 단계적 지원 구조**(Articulation→Coaching→Scaffolding→Modeling). Socratic은 CAM의 Articulation 단계에서 가장 강하게 작동. (`07_기술구현_상세.md` §5-4) |
| "데이터 윤리는 어떻게 다루나요?" | 본 발표는 합성 데이터로 시연합니다. 실 도입 시점에는 `docs/10_데이터윤리.md`에 따라 (1) 4분류 세분화 동의, (2) 메타인지 신호 학습자 노출 금지, (3) 외부 LLM 학습 송신 금지, (4) 열람·정정·삭제권 보장, (5) 미성년자 추가 보호. IRB 심의 별도 진행. |
